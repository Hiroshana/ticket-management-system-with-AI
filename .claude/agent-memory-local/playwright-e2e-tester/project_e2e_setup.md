## Directory layout

```
e2e/
  .gitignore              # ignores .auth/ — never commit session state
  .auth/
    admin.json            # saved admin session (generated by setup project)
    agent.json            # saved agent session (generated by setup project)
  pages/
    LoginPage.ts          # Page Object for /login
  setup/
    auth.setup.ts         # saves admin + agent storageState to .auth/
  auth.spec.ts            # comprehensive auth tests (31 tests across 6 groups)
playwright.config.ts      # root of monorepo — testDir: ./e2e
tsconfig.json             # root tsconfig covers playwright.config.ts + e2e/**/*.ts
```

## Playwright config key decisions

- `"module": "commonjs"` in root tsconfig — use `__dirname`, NOT `import.meta.dirname`
- Two projects: `"setup"` (runs auth.setup.ts) and `"chromium"` (depends on "setup")
- `workers: 1`, `fullyParallel: false` — single shared test database
- webServer 1: `bun run src/index.ts` in `./server`, env `NODE_ENV: test`
- webServer 2: `bun run dev` in `./client`
- `reuseExistingServer: !process.env.CI` — reuses locally running servers during dev
- screenshot on failure, video on retry

## Auth state paths

- Admin: `e2e/.auth/admin.json`
- Agent: `e2e/.auth/agent.json`

To use in a test file:
```ts
test.use({ storageState: path.join(__dirname, '../.auth/admin.json') });
```

## Run commands

```bash
bun run test:e2e        # create+migrate+seed test DB, then run all tests
bun run test:e2e:ui     # same but with Playwright UI
```

Or directly: `bunx playwright test`

## Prerequisites

- Test DB must be seeded (script does this automatically via `db:seed:test`)
- Both servers must be up OR `reuseExistingServer: false`
